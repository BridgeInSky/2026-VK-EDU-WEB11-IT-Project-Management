# 2026-VK-EDU-WEB11-IT-Project-Management
Название проекта: SmartHome Hub (Центр умного дома)
=

Elevator Pitch
-
> Люди хотят "умный дом", но сталкиваются с выбором из 100 брендов, которые не дружат друг с другом, и не могут найти мастера, который согласится на мелкий заказ (просто поставить розетку). Магазины техники теряют покупателя на этапе "А кто мне это подключит?".
SmartHome Hub — это B2B2C-экосистема. Мы не берем комиссию с продаж устройств. Мы продаем подписку на "Мозги" (облачный контроллер, объединяющий все протоколы Zigbee/Matter/Wi-Fi) и продвижение для мастеров и магазинов. Пользователь покупает лампочку у любого партнера, активирует подписку за 5$ в месяц, и через нашу платформу заказывает мастера, который за 15 минут "оживит" систему. Магазины получают сервисный отдел без найма, мастера — поток заказов, а пользователи — реально работающий умный дом без головной боли

### 1. Роли и пользовательские истории

Выделено 4 ключевых роли платформы SmartHome Hub.

| ID | Роль | История (User Story) |
|----|------|----------------------|
| US1 | Покупатель (C) | Я хочу авторизоваться по номеру телефона в один клик, чтобы не запоминать пароль. |
| US2 | Покупатель (C) | Я хочу видеть в каталоге, какие устройства работают с экосистемой SmartHome Hub, чтобы понимать, что купленное устройство я смогу подключить к своему «мозгу» и управлять из одного приложения |
| US3 | Покупатель (C) | Я хочу нажать кнопку «Мастер рядом» на карточке любого товара, чтобы сразу вызвать установщика без поиска по сторонним сайтам. |
| US4 | Покупатель (C) | Я хочу создать сценарий «Если датчик движения сработал → включить свет в коридоре», даже если датчик и лампочка — разных брендов. |
| US5 | Покупатель (C) | Я хочу оплатить заказ на установку через приложение с защитой (деньги заморожены до завершения работы), чтобы не бояться мошенников. |
| US6 | Мастер | Я хочу создать профиль за 5 минут (ФИО, фото, зона выезда, сертификаты) и начать получать заказы без стартовых взносов. |
| US7 | Мастер | Я хочу получать push-уведомления о новых заявках в моём районе с ценой и расстоянием, чтобы не пропускать заказы. |
| US8 | Мастер | Я хочу выкупить «Приоритетный показ» на 7 дней за 300 ₽, чтобы моя анкета была выше в выдаче у клиента. |
| US9 | Магазин (B2B) | Я хочу загрузить прайс-лист из 200 товаров CSV-файлом и указать свою наценку, чтобы не вводить каждую позицию вручную. |
| US10 | Магазин (B2B) | Я хочу установить на свой сайт белый виджет «Вызвать мастера SHH», чтобы клиенты не возвращали товар из-за сложности установки. |
| US11 | Админ | Я хочу модернировать мастеров: проверять паспорт и ИНН до первого заказа, чтобы отсеять мошенников. |

### 2. Приоритизация функционала (MoSCoW)

| Категория | Истории | Обоснование |
|-----------|---------|-------------|
| **Must have** (критично для запуска) | US1, US2, US3, US5, US6, US7, US11 | Без регистрации, каталога, вызова мастера и защиты от мошенников продукт не работает. |
| **Should have** (важно, но потерпит 2-4 недели) | US4, US8, US10 | Сценарии можно добавить позже, приоритетный показ — монетизация, но не критична для MVP. |
| **Could have** (усилит продукт) | US9, US12 | Масс-загрузка CSV и автопостроение маршрутов — для масштабирования, не для первой версии. |
| **Won't have** (исключено из MVP) | Интеграция с голосовыми ассистентами, свой протокол умного дома | Это feat, который требует много ресурсов. Откладываем на MLP. |

### 3. MVP и MLP

#### MVP 
MVP запускается через 2 месяца разработки и проверяет гипотезы: "Люди готовы платить за удобный вызов мастера для умного дома".

| Компонент | Что входит в MVP |
|-----------|------------------|
| **Каталог устройств** | 50 товаров от 3 партнёров (ручной импорт). Фильтры: по цене, по протоколу (Zigbee/Wi-Fi). |
| **Авторизация** | По номеру телефона (SMS-код). Без соцсетей. |
| **Биржа мастеров** | Профиль мастера (имя, рейтинг, зона выезда, цена за час). Верификация через фото паспорта (ручная модерация). |
| **Заказ установки** | Кнопка «Мастер рядом» на карточке товара. Оплата 20% предоплаты через карту (заморозка до завершения). |
| **Уведомления** | Push/TG-уведомление мастеру о новом заказе. Кнопки «Принять» / «Отказаться». |
| **Отзывы** | 5-звёздочная система после завершения заказа. Обязательное поле «Что понравилось / не понравилось». |

#### MLP

Через 3-4 месяца после MVP добавляем функции, за которые пользователи полюбят продукт:

| Категория | Что добавляем |
|-----------|---------------|
| **Сценарии («мозги»)** | Конструктор «Если → То» для любых устройств разных брендов. Бесплатный тариф: 3 сценария, до 10 устройств. |
| **Платная подписка** | $5/мес: безлимит устройств, 50 сценариев, удалённый доступ, приоритетная техподдержка. |
| **Приоритетный показ мастеров** | Платное поднятие анкеты в топ (на 1, 7, 30 дней). |
| **Виджет для магазинов** | Белый iframe-виджет «Вызвать мастера SHH» для вставки на сайт партнёра. |
| **Масс-загрузка CSV** | Магазины загружают тысячи товаров за 1 файл. |

### 4. Детализация требований (2 главные истории из MVP)
